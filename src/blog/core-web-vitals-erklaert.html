Core Web Vitals: Was Google wirklich von deiner Website will

LCP, FID, CLS – drei Abkürzungen, die über dein Google-Ranking entscheiden. Hier erfährst du, was sie bedeuten und wie du sie optimierst.

Seit 2021 sind die Core Web Vitals ein offizieller Google-Ranking-Faktor. Das bedeutet: Wenn deine Website langsam ist oder springende Inhalte hat, kann das dein Ranking negativ beeinflussen – selbst wenn dein Content erstklassig ist.

In diesem Artikel erkläre ich dir die drei Core Web Vitals so, dass du sie wirklich verstehst. Ohne Marketing-Buzzwords, ohne unnötige Komplexität.

LCP – Largest Contentful Paint

Was es misst: Wie lange dauert es, bis der größte sichtbare Inhalt (meist ein Bild oder eine Überschrift) geladen ist?

LCP Bewertung:

Gut: unter 2,5 Sekunden
Verbesserungswürdig: 2,5 - 4 Sekunden
Schlecht: über 4 Sekunden

Typische Probleme:

  • Große, unkomprimierte Bilder
  • Langsamer Server (billiges Hosting)
  • Blockierendes JavaScript und CSS
  • Lazy Loading für Above-the-Fold Content

So verbesserst du LCP:

  • Bilder in WebP-Format konvertieren und komprimieren
  • Hero-Bild mit preload vorladen
  • Schnelles Hosting wählen (nicht den billigsten Anbieter)
  • Kritisches CSS inline einbinden

FID – First Input Delay (jetzt INP)

Was es misst: Wie lange dauert es, bis die Website auf deine erste Interaktion (Klick, Tippen) reagiert?

Hinweis: Google hat FID im März 2024 durch INP (Interaction to Next Paint) ersetzt. INP misst alle Interaktionen, nicht nur die erste.

INP Bewertung:

Gut: unter 200 Millisekunden
Verbesserungswürdig: 200 - 500 ms
Schlecht: über 500 ms

Typische Probleme:

  • Zu viel JavaScript, das den Browser blockiert
  • Third-Party Scripts (Analytics, Chat-Widgets, Social Media)
  • Große JavaScript-Bundles

So verbesserst du INP:

  • JavaScript minimieren und aufteilen
  • Third-Party Scripts asynchron laden oder entfernen
  • Lange JavaScript-Tasks aufbrechen
  • Web Workers für schwere Berechnungen nutzen

CLS – Cumulative Layout Shift

Was es misst: Wie stark verschieben sich Elemente auf der Seite während des Ladens?

Du kennst das: Du willst auf einen Button klicken, aber plötzlich lädt ein Bild und alles springt nach unten. Das ist CLS – und es nervt Nutzer massiv.

CLS Bewertung:

Gut: unter 0,1
Verbesserungswürdig: 0,1 - 0,25
Schlecht: über 0,25

Typische Probleme:

  • Bilder ohne definierte Größe (width/height)
  • Werbung und Embeds ohne reservierten Platz
  • Web Fonts, die zu Flash of Unstyled Text (FOUT) führen
  • Dynamisch eingefügte Inhalte (Cookie-Banner, Newsletter-Popups)

So verbesserst du CLS:

  • Immer width und height für Bilder angeben
  • Platzhalter für Ads und Embeds reservieren
  • font-display: swap für Web Fonts
  • Animationen mit transform statt Position/Größe

So misst du deine Core Web Vitals

Es gibt mehrere kostenlose Tools, um deine Werte zu prüfen:

  1. PageSpeed Insights (pagespeed.web.dev) – Das offizielle Google-Tool
  2. Google Search Console – Zeigt echte Nutzerdaten
  3. Lighthouse – In Chrome DevTools integriert
  4. WebPageTest – Für detaillierte Analysen

Wichtig: Teste immer auf Mobile! Die meisten deiner Besucher kommen wahrscheinlich vom Smartphone, und dort sind die Werte oft schlechter.

Schnelle Checkliste

Core Web Vitals Checkliste:

  • Bilder komprimiert und in WebP
  • Alle Bilder haben width/height
  • Hero-Bild wird vorgeladen
  • JavaScript ist minimiert
  • Third-Party Scripts sind asynchron
  • Fonts nutzen font-display: swap
  • Hosting ist schnell genug
  • Mobile Performance getestet

Fazit

Core Web Vitals sind kein Hexenwerk. Die meisten Probleme lassen sich mit ein paar grundlegenden Optimierungen lösen. Das Wichtigste:

  • Bilder optimieren (löst meist LCP und CLS)
  • Weniger JavaScript (löst INP)
  • Gutes Hosting (hilft bei allem)

Oder du lässt dir eine handcodierte Website erstellen, die von Anfang an auf Performance optimiert ist. Unsere Websites erreichen standardmäßig 95+ Punkte bei PageSpeed Insights.
